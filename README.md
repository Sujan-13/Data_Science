# Data Science 📊🐍

A collection of exploratory data analysis, visualization, and machine learning mini-projects built while learning data science with Python. Each folder is a self-contained notebook (or set of notebooks) covering a specific dataset, library, or statistical concept.

## 📁 Repository Structure

```
Data_Science/
├── Day72_college_majors/           # Analyzing college majors vs earnings
├── Day73_programming_languages/    # Programming language popularity trends
├── Day77_Numpy/                    # NumPy fundamentals & array operations
├── Day78_sns/                      # Data visualization with Seaborn
├── Day79_novelprize/               # Nobel Prize winners analysis
├── Day80_handwashing/              # Semmelweis handwashing discovery analysis
├── Day81_/                         # Multivariable regression & home valuation model
├── Google Trends Data Viz (start)/ # Google Trends data visualization
├── LEGO Notebook and Data (start)/ # LEGO dataset exploration
└── playstore/                      # Google Play Store app market analysis
```

## 🗂️ Project Summaries

| Project | Description | Key Skills |
|---|---|---|
| **Day72 – Programming/College Data** | Explores `day72_college.csv` to compare college majors, gender splits, and median earnings | Pandas, Matplotlib, data cleaning |
| **Day73 – Programming Languages** | Analyzes `day73_programming.csv` (Stack Overflow-style survey data) to chart the rise and fall of programming language popularity over time | Pandas, Matplotlib, time-series line charts |
| **Day77 – NumPy** | Core NumPy exercises: array creation, indexing, broadcasting, and vectorized operations | NumPy fundamentals |
| **Day78 – Seaborn** | Statistical data visualization exercises using Seaborn (distribution plots, pair plots, heatmaps) | Seaborn, statistical plotting |
| **Day79 – Nobel Prize** | Explores Nobel Prize winners by country, category, and gender over the prize's history | Pandas, Plotly/Matplotlib, groupby analysis |
| **Day80 – Handwashing** | Recreates Dr. Semmelweis's famous discovery linking handwashing to reduced mortality rates using historical clinic data | Pandas, hypothesis-style analysis, time series |
| **Day81 – Multivariable Regression & Valuation Model** | Builds a linear regression model on `boston.csv` (Boston housing dataset) to estimate property values based on features like rooms, distance to employment centers, and pupil-teacher ratio | Scikit-learn / Statsmodels, multivariable regression, log transformation, R² evaluation |
| **Google Trends Data Viz** | Visualizes Google Trends search interest data over time for various keywords | Pandas, Matplotlib, time-series analysis |
| **LEGO Notebook and Data** | Explores historical LEGO set data — themes, piece counts, and set releases over the decades | Pandas, Matplotlib, merging datasets |
| **Play Store App Analysis** | Analyzes the Google Play Store dataset to explore app categories, ratings, installs, and pricing trends | Pandas, Plotly, data cleaning, outlier handling |

## 🛠️ Tech Stack

- **Language:** Python 3.x
- **Core Libraries:** NumPy, Pandas
- **Visualization:** Matplotlib, Seaborn, Plotly
- **Modeling:** Scikit-learn, Statsmodels
- **Environment:** Jupyter Notebook

## 🚀 Getting Started

1. Clone the repository
   ```bash
   git clone https://github.com/<your-username>/Data_Science.git
   cd Data_Science
   ```

2. Create a virtual environment and install dependencies
   ```bash
   python -m venv venv
   source venv/bin/activate      # On Windows: venv\Scripts\activate
   pip install numpy pandas matplotlib seaborn plotly scikit-learn statsmodels jupyter
   ```

3. Launch Jupyter and open any project notebook
   ```bash
   jupyter notebook
   ```

4. Navigate into the relevant `Day##_*` or named folder and run the `.ipynb` file. Each notebook reads its dataset from a `.csv` file located in the same folder.

## 📌 Notes

- Folders suffixed with `(start)` contain the starter/incomplete version of a notebook meant to be filled in as an exercise.
- `.ipynb_checkpoints` folders are auto-generated by Jupyter and can be safely ignored or added to `.gitignore`.
- Each project is independent — no shared dependencies between folders beyond the common Python libraries listed above.
